# TAS - Telegram Anti-spam System

## Обзор системы

TAS - это автоматизированная система для обнаружения и управления спамом в группах Telegram. Система использует комбинацию методов, включая кэширование, обнаружение очевидного спама, анализ на основе GPT и мнения модераторов для принятия решений о спам-сообщениях.

## Основные компоненты

1. Telegram клиент для взаимодействия с API Telegram
2. База данных PostgreSQL для долгосрочного хранения данных
3. Redis для кэширования и быстрого доступа к данным
4. Интеграция с GPT для анализа сообщений
5. Система обработки мнений модераторов
6. Механизмы автоматического обнаружения спама

## Типы сообщений

1. checkMsg - сообщения для классификации
2. sysMsg - системные сообщения
3. addMsg - дополнительные сообщения

## Процесс обработки отчета

1. Проверка кэша
2. Проверка на очевидный спам
3. Проверка мнений модераторов
4. GPT анализ
5. Принятие окончательного решения
6. Сохранение результата

## Автоматические критерии определения спама

1. Чрезмерное использование эмодзи или необычное форматирование текста
2. Повторяющиеся элементы в сообщении
3. Множественные ссылки и контактная информация
4. Чрезмерное использование заглавных букв
5. Подозрительная структура сообщения
6. Наличие медиа, ссылок или контактной информации в сообщении с более чем 2 жалобами
7. Наличие содержимого типа "story" в сообщении
8. Наличие потенциально опасных типов файлов

## Правила принятия решений (после обработки мнений модераторов)

- 2 или более "– Flood" -> SPAM (Reason.MODERATOR_2_FLOOD)
- 2 или более "– Not Spam" -> NOT SPAM (Reason.MODERATOR_2_NOT_SPAM)
- 1 "– Flood" и 0 "– Not Spam" -> SPAM (Reason.MODERATOR_1_FLOOD)
- 1 "– Not Spam" и 0 "– Flood" -> NOT SPAM (Reason.MODERATOR_1_NOT_SPAM)
- 1 "– Flood" и 1 "– Not Spam" -> GPT
- Нет мнений -> Используется оценка GPT
- Любая другая комбинация -> "/undo" (с логированием неожиданной ситуации)

## Интеграция с GPT

- Выбор модели GPT в зависимости от длины сообщения
- Анализ сообщения с учетом контекста (количество жалоб, название группы, отправитель)
- GPT возвращает два числа: классификацию (0 для не-спама или 1 для спама) и уровень уверенности (от 0 до 100)

## Система автоматического обновления списка ключевых слов для спама

- Ежедневное обновление списка топ-100 спам-фраз
- Анализ спам-сообщений и не-спам сообщений за последние 30 дней
- Использование метода N-грамм (триграмм) для выделения фраз
- Вычисление взвешенной оценки для каждой фразы

## Процесс возврата к предыдущим отчетам

- Хранение идентификаторов последних 10 обработанных отчетов
- Механизм отправки команды "/undo" с идентификатором отчета
- Автоматическая пауза и возобновление работы бота при необходимости

## Команды администратора

- "/start" - активация автоматического режима
- "/stop" - остановка работы приложения и переход в ручной режим
- "/time [миллисекунды]" - установка задержки обработки сообщений
- "/delay [миллисекунды]" - установка задержки между командами
- "/status" - получение текущего статуса бота
- "/db" - запрос экспорта базы данных в CSV
- "/toggle [obvs/cache/gpt/mods]" - включение/выключение отдельных этапов проверки
- "/correct #rXXXXX" - исправление решения для конкретного отчета
- "/reset" - очистка кэша Redis
- "/memory" - получение информации об использовании памяти
- "/gc" - принудительный вызов сборщика мусора

## Переменные окружения

- `DEEP_LOG`: Включение детального логирования (true/false)
- `PORT`: Порт для сервера Express
- `ADMIN_ID`: ID администратора в Telegram
- `DATABASE_URL`: URL подключения к PostgreSQL
- `REDIS_URL`: URL подключения к Redis
- `API_ID`: API ID для Telegram
- `API_HASH`: API Hash для Telegram
- `SESSION_STRING`: Строка сессии для Telegram клиента
- `BOT_ID`: ID бота в Telegram
- `BOT_ACCESS_HASH`: Access Hash бота в Telegram
- `DB_CHECK_INTERVAL`: Интервал проверки базы данных (в миллисекундах)
- `HEALTH_CHECK_INTERVAL`: Интервал проверки здоровья системы (в миллисекундах)
- `PROCESSING_INTERVAL`: Интервал обработки отчетов (в миллисекундах)
- `OPENAI_API_KEY`: Ключ API для доступа к сервисам OpenAI
