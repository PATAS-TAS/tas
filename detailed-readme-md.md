# Система проверки спама в Telegram

## Описание проекта

Данный проект представляет собой комплексную систему для обнаружения и фильтрации спама в Telegram. Система использует многоуровневый подход к анализу сообщений, включая проверку кэша, анализ очевидных признаков спама, использование GPT для глубокого анализа контента и интеграцию с Google Cloud Vision API для анализа изображений.

## Основные компоненты

- **GramJS**: Используется для взаимодействия с API Telegram
- **Express**: Веб-сервер для обработки запросов
- **OpenAI API**: Используется для продвинутой классификации сообщений с помощью GPT
- **Google Cloud Vision API**: Применяется для анализа изображений
- **Redis**: Используется для кэширования результатов проверки

## Установка и настройка

### Предварительные требования

- Node.js (версия 18 или выше)
- Yarn (версия 1.x)
- Redis сервер
- Доступ к API Telegram, OpenAI и Google Cloud Vision

### Установка зависимостей

```bash
yarn install
```

### Настройка переменных окружения

Создайте файл `.env` в корневой директории проекта и добавьте следующие переменные:

```
API_HASH=your_telegram_api_hash
API_ID=your_telegram_api_id
PHONE_NUMBER=your_telegram_phone_number
BOT_ID=your_telegram_bot_id
ADMIN_ID=your_telegram_admin_id
OPENAI_API_KEY=your_openai_api_key
GOOGLE_CLOUD_PROJECT=your_google_cloud_project_id
REDIS_URL=your_redis_url
PORT=3000
CACHE_TTL=86400
```

## Архитектура системы

Система построена на основе паттерна "Цепочка обязанностей" и включает следующие основные компоненты:

1. **Инициализация клиента**: Настройка и подключение к Telegram API.
2. **Обработка входящих сообщений**: Сообщения добавляются в буфер для последующей проверки.
3. **Процесс проверки**: 
   - Проверка кэша
   - Анализ очевидных признаков спама
   - Глубокий анализ с использованием GPT
   - Анализ изображений с помощью Google Cloud Vision API
4. **Обработка результатов**: Определение спама и отправка соответствующего ответа.
5. **Кэширование результатов**: Сохранение результатов проверки в Redis для оптимизации производительности.

## Основные функции и их описание

### 1. Инициализация клиента (`initClient`)

- Устанавливает соединение с Telegram API
- Обрабатывает процесс авторизации, если необходимо
- Сохраняет строку сессии для последующих подключений

### 2. Обработка сообщений

- `checkMsg`: Обрабатывает входящие сообщения для проверки
- `handleNextReport`: Обрабатывает сообщения о следующем отчете
- `sysMsg`: Обрабатывает системные сообщения с информацией о жалобах

### 3. Проверка спама

- `checkCache`: Проверяет наличие результата в кэше
- `checkObvious`: Анализирует очевидные признаки спама
- `checkGPT`: Выполняет глубокий анализ с использованием GPT

### 4. Анализ медиа-контента

- `preprocessAndAnalyze`: Подготавливает сообщение и анализирует медиа-контент
- `analyzeMediaMessage`: Анализирует медиа-сообщение с помощью Google Cloud Vision API

### 5. Управление кэшем

- `saveToCache`: Сохраняет результат проверки в кэш
- `getFromCache`: Получает результат из кэша
- `checkCacheUsage`: Проверяет использование кэша и при необходимости очищает старые записи

### 6. Процесс восстановления

- `startRecovery`: Запускает процесс восстановления в случае ошибок или зависаний
- `resetRecoveryTimers`: Сбрасывает таймеры восстановления

## Параметры конфигурации

- `MAX_FILE_SIZE`: Максимальный размер файла для анализа (5 МБ)
- `REDIS_POOL_SIZE`: Размер пула соединений Redis (5)
- `MAX_CACHE_USAGE`: Максимальное использование кэша (90%)
- `MAX_PROCESSING_TIME`: Максимальное время обработки сообщения (59 секунд)
- `CHECK_MSG_TIMEOUT`: Таймаут для проверки наличия новых сообщений (55 секунд)

## Команды администратора

- `/time <секунды>`: Устанавливает интервал обработки сообщений
- `/start`: Включает автоматический режим
- `/stop`: Выключает автоматический режим
- `/toggle <feature>`: Включает/выключает определенную функцию (vision, cache, obvious, gpt, mod)

## Особенности и оптимизации

1. **Многоуровневая проверка**: Система использует несколько уровней проверки для повышения точности определения спама.
2. **Кэширование**: Результаты проверок кэшируются для оптимизации производительности при повторных проверках.
3. **Анализ изображений**: Интеграция с Google Cloud Vision API позволяет анализировать содержимое изображений.
4. **Гибкая настройка**: Администратор может включать/выключать отдельные компоненты системы.
5. **Механизм восстановления**: Реализован механизм для обработки ошибок и зависаний.
6. **Оптимизация использования API**: Реализована система повторных попыток и таймаутов для работы с внешними API.

## Условия определения спама

- Высокое количество жалоб
- Наличие URL или подозрительных фраз в имени отправителя
- Наличие спам-фраз в тексте сообщения
- Чрезмерное использование определенных эмодзи
- Наличие дублирующихся или рекламных ссылок
- Подозрительные шаблоны сообщений
- Избыточная контактная информация
- Дублирующиеся медиафайлы или файлы с потенциально вредоносными расширениями
- Высокая оценка вероятности спама от GPT (с динамическим порогом)
