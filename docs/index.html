API Migration Guide

We are introducing a new response schema alongside the legacy fields to enable a smooth migration.

Timeline

  • Deprecation header: Deprecation: true
  • Sunset header: Sunset: <YYYY-MM-DD> (6 months from first release)
  • Link header: Link: <https://kiku-jw.github.io/tas/#migration>; rel="deprecation"
  • Legacy fields removal: after Sunset date.

New Response Schema

POST /v1/classify now returns BOTH formats:

  • New: spam, score, reasons[] (with code, text, weight), path, request_id
  • Legacy (to be removed after Sunset): is_spam, confidence, reason

Example Response

{
  "spam": true,
  "score": 0.91,
  "reasons": [
    {"code": "commercial_trade_offer", "text": "Commercial trade offer", "weight": 0.4}
  ],
  "path": "rules",
  "request_id": "r_01ab234cdef",
  "is_spam": true,
  "confidence": 0.91,
  "reason": "Commercial trade offer"
}

Migration Tips

  • Prefer spam, score, reasons, path, request_id.
  • Keep reading legacy fields during the transition window.
  • Use /v1/batch for efficient batch classification.
