Модульная структура проекта:

1. Настройки:
1.1 импорты, конфиги, 
1.2 интерфейсы, классы, бд строки
1.3. инициализация Telegram, Redis, Postgre, Google Vision

2. Прием сообщений:
2.1 от бота проверяемые (и типы медиа) - добавляем в буффер
2.2 от бота системные сообщения (и значения) - добавляем в буффер
2.3 от админа команды

3. Быстрые проверки:
3.1 проверка совпадения с горячим и холодным кешем и ответ как в кеше (окончательное решение)
3.2 проверка на топ спам фраз (окончательное решение)
3.3 проверка на признаки очевидного спама (окончательное решение)

4. Подготовка к глубоким проверкам: 
4.1 Оценка подозрительных моментов, паттернов, похожие на топ спам фразы (даёт только оценку для следующих проверок)
4.2 Оценка с помощью моделей таких как text-davinci-003, text-embedding-ada-002, text-moderation и локального машинного обучения (даёт только оценку для следующих проверок)
4.2 Препроцессинг текста, препроцессинг медиа с помощью Google Vision (текст с медиа добавляем в буффер и проверяем как текст сообщения, только подготавливает для следующих проверок)

5. Глубокие проверки:
5.1 проверка с помощью LLM - файтюнинговая модель GPT. Модель, промт, макс токенов, температура - все значения меняются в зависимости от оценки, языка и т.д. (окончательное решение только если высокое значение уверенности)
5.2 проверка неясных сообщений с помощью мнения модераторов, например ссылки и медиа с 1-2 жалобами (окончательное решение, если есть мнение модераторов)
5.3 проверка медиа с помощью Claude Sonnet 3.5, проверка текста с помощью LLama 3.0 405B. Если решение отличается от проверки LLM GPT - третья модель тоже проверяет. И тогда на основании мнения большинства (двух моделей из трех) делается окончательное решение. Если никакого текста для проверки так и не дошло, вывод делается на основании имени отправителя и предыдущих оценок. Если имени и оценок нет, отправляем /undo

6. Результат:
6.1 цепочка проверок
6.2 отправка ответа
6.3 запись в кеш и бд для файтюнинга GPT, машинного обучения и топ спам фраз

7. Обработка ошибок и управление:
7.1 логи, тесты
7.2 восстановление из разных ситуаций ошибок
7.3 команды и уведомление админа
